Руководство по использованию ПО Crystal в среде ВЦ¶

Общее описание¶

CRYSTAL - программа для проведения неэмпирических квантово-химических расчетов основного состояния периодических и молекулярных систем в рамках метода Hartree Fock и теории функционала плотности, используя атомный базисы на основе гауссовых функций. Код применяется в исследованиях физических и химических свойств молекул, полимеров, поверхностей и объемных кристаллических твёрдых тел: геометрической структуры, колебательных и упругих свойств, электронного строения, магнитных и диэлектрических свойств.

Поддерживаемые версии и условия использования ПО¶

Лицензионные ограничения¶

Имеющиеся версии Crystal допустимо использовать только сотрудникам и студентам СПбГУ и только для академических целей. При публикации результатов научной работы, полученных с помощью ПО Crystal, обязательно указание ссылки согласно официальным требованиям. Для получения доступа к необходимой версии Crystal необходимо подать заявку на исследование через систему Научного Парка СПбГУ.

Доступные версии и варианты использования¶

В Вычислительном Центре доступна наиболее актуальная версия Crystal 17 1.0.2 (19.04.2018), а также более ранние версии вплоть до Crystal 09 1.0.1. Данная инструкция применима только для версии 17 1.0.2 и 1.0.1 - по вопросам использования более ранних версий, обращайтесь в тех. поддержку Вычислительного центра. Расчеты с Crystal 17 поддерживаются в параллельном и однопоточном режиме на всех вычислительных кластерах центра и на ВМ на базе Linux CentOS 6 и 7. Перед использованием Crystal, ознакомьтесь с инструкцией по удаленному подключению к виртуальной машине на базе Linux.

Описываемый в данной инструкции вариант использования ПО главным образом нацелен на упрощение работы с программой в среде Вычислительного Центра, чтобы Вы могли сконцентрироваться на научной стороне задачи и не беспокоиться об аппаратной конфигурации того или иного вычислителя. При обнаружении каких-либо технических проблем с использованием ПО (нехватка оперативной памяти, диска и т.п) обращайтесь в тех. поддержку Вычислительного Центра. Если у Вас есть рекомендации по более оптимальной настройке ПО или иные советы по упрощению работы, так же обращайтесь в тех. поддержку, мы рассмотрим Ваши предложения.

Руководство по использованию Crystal 17¶

Запуск расчетов

Программа Crystal разделена на две подпрограммы: Crystal и Properties. Обе можно запускать как на виртуальной машине, так и на вычислительных кластерах на любом количестве ядер (не рекомендуется использовать более 72).

Для запуска Crystal используйте скрипт submit-crystal, а для Properties - submitproperties. Кроме того, если в рамках одного расчета требуется провести как основной расчет в Crystal, так и расчет свойств в Properties, то для этого при запуске скрипта submit-crystal укажите ключ -P (см. подробнее Параметры запуска). Оба скрипта находятся в директории /usr/local/hpc/crystal.

Чтобы не набирать в командной строке каждый раз полный путь к скрипту запуска, Вы можете добавить в .bashrc-файл в своей домашней директории на виртуальной машине следующую директиву:

Для Crystal 17 1.0.2:

source /usr/local/hpc/etc/crystal/17/1.0.2.bashrc

Для Crystal 17 1.0.1:

source /usr/local/hpc/etc/crystal/17/1.0.1.bashrc

Если по каким-либо причинам Вам требуется использование и той и другой версии программы в рамках одного сеанса, то вышеуказанные директивы можно вызывать в командной строке перед непосредственным вызовом скриптов запуска submit-crystal и submit-properties, а в bashrc-файле можно указать лишь наиболее часто используемую версию.

Предупреждение

Если вышеприведенные директивы вообще не использовать, то при запуске скриптов будет всегда использоваться наиболее актуальная версия Crystal 17 1.0.2

Параметры запуска¶

При запуске скриптов без параметров в терминале будет отображена инструкция по их использованию. Параметры можно указывать в произвольном порядке.

Для скрипта submit-crystal доступны следующие параметры:

  • <input-file>.d12
    Обязательный параметр. Имя входного файла c указанием пути и расширения. Можно использовать полный формат пути, например: /home/username/test.d12, при этом можно использовать переменные окружения, например: $HOME/test.d12. Также можно указывать путь относительно текущей директории (используйте команду pwd, чтобы узнать текущую директорию), например: ../test.d12 - нужный файл находится на уровень выше текущей директории. Чтобы избежать ошибок, рекомендуется сначала перейти в директорию, где находится нужный входной файл (команда cd) и тогда при запуске скрипта путь к входному файлу можно не указывать вовсе.

  • [-m <machine>]
    Опциональный параметр. Тип машины, на которой будет запущен расчет (необходимо иметь соответствующий доступ): vm - виртуальная машина, hw - кластер Huawei, tp - кластер Tплатформ, hp - гибридный кластер Hewlett-Packard , smp - SMP кластера
    Значение по умолчанию - hw

  • [-q <queue>]
    Опциональный параметр. Очередь на кластере, в которой нужно запустить задачу (см. список очередей)
    Значение по умолчанию - long

  • [-с <n_cores>]
    Опциональный параметр. Число ядер CPU. Рекомендации по оптимальному выбору процессорных ядер читайте в официальной документации к программе. Эффективность распараллеливания сильно зависит от использованного метода расчета и размера системы.
    Значение по умолчанию - 2

  • [-T]
    Опциональный параметр. При использовании данного ключа временные файлы во время расчета будут храниться в рабочей директории проекта на виртуальной машине. Указание данного ключа требуется только в некоторых особых случаях и по умолчанию используется только при расчете на самой виртуальной машине. Для более подробной информации смотрите раздел label

  • [-B] Опциональный параметр. При указании данного ключа при запуске будет создана директория, в которой будут сохранены все файлы в рабочей директории, имя которых совпадает с именем входного d12-файла. Это может быть полезно в случаях, когда требуется сохранить данные предыдущих расчетов.

Примеры запуска расчета¶

Чтобы, например, запустить на расчет входной файл test.d12 на виртуальной машине, а затем рассчитать свойства системы согласно данным в файле test.d3, выполните следующую команду:

submit-crystal $HOME/crystal/test.d12 -c 2 -m vm -P

Если расчет ресурсоемкий, то можно запустить его, например, на кластере Huawei, указав ключи -m hw и -q long (очередь long используется по умолчанию):

submit-crystal -c 12 -m hw -q long $HOME/crystal/test.d12

А для расчета свойств отдельно от основного расчета, выполните эту команду:

submit-properties $HOME/crystal/test.d3 test.f9 -c 2 -m vm

Перезапуск расчетов¶

Необходимые требования к перезапуску расчетов Crystal и сама инструкция по перезапуску задач приведены в официальной документации к программе.

Согласно данной инструкции для перезапуска прерванных расчетов требуются разные файлы. При штатном завершении программы они будут записаны в рабочую директорию по окончании расчета. Однако если расчет прервется по технической причине, или чаще всего при превышении предельного времени счета в очереди, то все файлы, кроме выходного out-файла будет утеряны и возможности возобновления расчетов будет ограничены. Если велик риск обрыва расчета по причине нехватки расчетного времени в очереди на кластере, то рекомендуется запускать Crystal с ключом -T. В этом случае временные файлы расчета будут сохраняться в рабочую директорию. Однако, в этом случае за счет более низкой производительности файловой системы, время расчета может увеличиться, поэтому не рекомендуется использовать этот ключ без особой причины.

Чтобы воспользоваться данными из предыдущих расчетов в новом, нужно будет при запуске скриптов указать ключ -R.

Использование XCrysDen¶

Для подготовки входных файлов Crystal, а также для визуализации и анализа результатов можно использовать программу XCrysDen.

Для того чтобы воспользоваться XCrysDen на виртуальной машине Linux, необходимо подключится к машине в режиме поддержки графического интерфейса, используя протокол ssh -x (воспользуйтесь инструкцией по удаленному подключению к виртуальной машине на базе Linux, раздел “Запуск графических приложений”).

Перед первым использованием программы выполните следующие команды в командной строке:

source /usr/local/hpc/etc/xcrysden/1.5.60.bashrc

prepare_xcrysden

После этого можно запускать программу из командной строки:

xcrysden

После запуска программа будет выводить в терминал служебную информацию, которая может быть полезна при возникновении проблем. Однако при этом терминал будет занят, и Вы не сможете выполнять другие команды из него, поэтому для использования XCrysDen рекомендуется либо создавать отдельный сеанс, либо запускать программу в фоне (xcrysden &).

Если возникли проблемы¶

Если у Вас возникли проблемы при использовании ПО, обращайтесь в тех. поддержку Вычислительного Центра по адресу support@cc.spbu.ru.

Опишите в письме саму проблему и условия ее возникновения, а так же приложите по возможности входной файл задачи, выходной out-файл, а также, если расчет производится на кластере, то и файлы вывода системы очередей: <имя задачи>.e<job-id> и <имя задачи>.o<job-id>, создающиеся в рабочей директории после завершения задачи.

Если проблемы вызваны на стороне Вычислительного Центра, то они решаются согласно внутреннему регламенту. Если проблема связана с самим ПО, то Ваше обращение перенаправляется в службу поддержки автора ПО.

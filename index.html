Создание проекта¶

Убедитесь, что у Вас установлен sphinx. На официальном сайте предложено много способов установки.

Создайте директорию для нового проекта и перейдите в нее в командной строке/терминале.
Для инициализации проекта необходимо выполнить команду sphinx-quickstart:

sphinx-quickstart

Программа задаст ряд вопросов. Все настройки можно будет позже изменить в файле conf.py.

После ответа на вопросы будут созданы файлы index.rst, conf.py, Makefile, _build, _static, _templates.

Makefile — содержит инструкции для генерации результирующего документа командой make.

_build — директория, в которую будут помещены файлы в определенном формате после того, как будет запущен процесс их генерации.

_static — в эту директорию помещаются все файлы, не являющиеся исходным кодом (например, изображения). Позже создаются связи этих файлов с директорией build.

conf.py — содержит конфигурационные параметры Sphinx, включая те, которые были выбраны при запуске sphinx-quickstart в окне терминала.

index.rst — это корень проекта. Он соединяет документацию воедино, если она разделена на несколько файлов.

В каталоге проекта находится файл index.rst, который служит для объединения всех файлов в один проект. .. перед абзацем означают комментарий. Также они служат объявлением многих команд. .. toctree:: объединяет отдельные файлы в единый проект. Содержимое index.rst не должно включать много информации и в нём обязательно должна присутствовать директива .. toctree::.

Чтобы включить в проект другие файлы, необходимо прописать названия этих файлов в .. toctree::. Для примера создадим в корне проекта файл example1.rst и example2.rst со следующим содержанием:

Это пример
==========

В index.rst это записывается следующим образом:

Оглавление:

.. toctree::
   :maxdepth: 2

   example1
   example2

Название файла пишется без расширения. Также важен отступ и пустая строка.

Генерация документа¶

Для генерации документации в HTML формат необходимо выполнить в командной строке команду make html. Произойдет сборка HTML, выходные файлы будут помещены в директорию _build/html/. Перейдите в неё и откройте файл index.html в браузере.

Если на странице вместо кириллицы неизвестные символы, то необходимо еще раз сохранить файл rst с указанием кодировки Utf-8.

Файл конфигурации conf.py позволяет настроить имя автора, название документа и многое другое.

Команду make.html необходимо выполнять после каждого сохраненного изменения файлов .rst или conf.py.

Здесь можно выбрать понравившуюся тему и установить ее. Чтобы применить ее к своему проекту, необходимо поменять некоторые данные в conf.py (на странице установки написано, что нужно вставить).

Синтаксис reStructuredText¶

  • Отступы и пробелы имеют значение для команд разметки, но не имеют значения внутри текста (10 пробелов будут отображены как один);

  • В командах (директивах) используется символ обратной кавычки «`», который располагается на клавише с буквой ё и символом ~. Использование обычных одинарных кавычек в командах не приведет к желаемым результатам.

Абзацы¶

Абзацы отделяются друг от друга пустой строкой:

Первый абзац...

Строки параграфов начинаются от левой границы
и отделяются параграфы друг от друга пустой строкой.

Чтобы сделать принудительный перенос строки, необходимо в начале rst-файла написать:

 .. |br| raw:: html

</br>

В конце строки, после которой требуется перенос, необходимо написать |br|.

Заголовки¶

Заголовки подчеркиваются (или отбиваются сверху и снизу) с помощью небуквенных и нецифровых 7-битных ASCII символов.

Заголовок 1 уровня
==================

Заголовок 2 уровня
------------------

Заголовок 3 уровня
~~~~~~~~~~~~~~~~~~

Заголовок 4 уровня
""""""""""""""""""

Отбивка должна быть не короче текста заголовка.

Начертание¶

**жирный текст**

*курсив текст*

Такое начертание текста достигается обособлением двумя обратными кавычками:

``Такое начертание текста``

Списки¶

Нумерованные списки создаются с помощью символа решетки с точкой #.:

#. Один
#. Два
#. Три

Или:
5. Пять
6. Шесть
#. Семь

Результат:

  1. Один

  2. Два

  3. Три

Или:
5. Пять 6. Шесть #. Семь

Маркированные списки создаются с помощью символа звездочки *. Пробелы после маркера обязательны:

* Один
* Два
* Три

Результат:
* Один * Два * Три

Вложенные списки

* Первый уровень
    * Второй уровень
        * Третий уровень

Результат:

  • Первый уровень
    • Второй уровень

      • Третий уровень
